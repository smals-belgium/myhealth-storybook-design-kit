      <td>Section within a list **about 5 to 15 values** logically ordered</td>
      <td>Selection within very long list of values</td>

    <tr>
      <td>
        **Characteristics**
        - Users can see all their choices right away
        - Can take up valuable space
        - Most of the time **with a default option** (Exception: Gender)
        - Always mandatory field
        - If more than 7 options consider using simple select or autocomplete
      </td>
      <td>
        **Characteristics**
        - Easier for **touch interfaces**
        - Users can see all their choices right away
        - Can contain text and / or icons
        - Takes less space than radio
        - For a very limited list of options (3-4)
      </td>
      <td>
        **Characteristics**
        - More complex for those who only use the keyboard
        - Takes less space
        - Users need to click to see the options
        - The label introducing the values has to be very explicit
        - **Default value = Select**
        - **10 Options and more **consider using **autocomplete**
      </td>
      <td>
        **Characteristics**
        - Autocomplete speeds up from completion by **filtering out the list while typing**
        - works well when the **user is familiar with the context** (e.g. list of cities or countries works well for everyone, a
        list of body areas works fine for caregivers)
      </td>
    </tr>
