# Material

> The My Health belgium.be design kit is built upon Angular Material, using the framework's extensive excellent accessibility and design features.
> Our design kit overrules Angular Material to push forward our own design, while taking advantage of these features.

## Color palette

The color palette used by MyHealth adheres to the [Material Design color system](https://m3.material.io/styles/color/system/overview), with a few custom colors added.

The colors improted into Angular Material are primary, secondary, tertiary, neutral and warning.
The success and error colors are not imported into Angular Material, as the system does not use these, though they are still used in the design kit.

All of these colors can be used in the design kit by calling the corresponding color variable of format `${color}-{shade}`.
For example: `$primary-40`.

These colors are used to build the design kit, and are available as variables in the design kit, however we **advise against using these variables directly** in your app.
Instead, **use the [Design Kit's color tokens](/docs/foundations-color-tokens--docs)**.

## Standard values

Because we want to stay within the standards of Angular Material as much as possible, we will use their standard values for our custom styling as well.
These standards can include heights of base components, breakpoints, etc.

### Breakpoints

One of the most important Material standards we use are the following breakpoints:

<Markdown>
  {`
| token                  | px   |
| :--------------------- | :--- |
| \`$media-breakpoint-xs\` | 600  |
| \`$media-breakpoint-s\`  | 960  |
| \`$media-breakpoint-m\`  | 1280 |
| \`$media-breakpoint-l\`  | 1920 |
`}
